Project Requirements & Design Proposal Webpage

💻 Project Overview: The UNCC HCI Interact Project Hub

📜 Application and Purpose

The web application, The UNCC HCI Interact Project Hub, is designed for the HCI Interact student group at UNC Charlotte. Its primary purpose is to serve as a centralized, dynamic platform to showcase past and ongoing student projects, provide essential information about the group, and encourage engagement and recruitment of new members. The site will improve the group's visibility within the university and serve as a portfolio for the students involved.

👤 Intended Users of the Website

  • Current HCI Interact Members: For referencing group information and project templates.
  • Prospective UNCC Students (Undergraduate and Graduate): Seeking student organizations and viewing examples of the group's work.
  • UNCC Faculty and External Industry Professionals: Visiting to view student portfolios, find collaborators, or recruit interns.

📰 Overview of the Content of the Website

  • About Us: Group mission, history, and executive board member profiles.
  • Projects: Detailed case studies of past and current projects.
  • Events: A dynamic calendar or list of upcoming meetings, workshops, and social events.
  • Resources: Links to relevant academic papers, design tools, and external HCI-related websites.
  • Join/Contact: A clear call-to-action page with a sign-up form and general contact information.

📋 Page Design (Detailed Requirements for Each Page)

1. Home Page (*index.html*)

  • Purpose: Provide an engaging introduction and direct users to key content.
  • Audience: All users.
  • Content: Hero section, mission statement, dynamically loaded featured projects.
  • Data Entry/Validation: No data entry fields.
  • Buttons/Hyperlinks/Drop Downs: Buttons for "View Projects" and "Join Now."
  • Actions: Dynamic loading of featured projects via AJAX on page load.

2. About Us (*about.html*)

  • Purpose: Detail the group's history, mission, and introduce the leadership team.
  • Audience: All users, especially Prospective Students.
  • Content: Textual history, executive board member images/profiles.
  • Data Entry/Validation: No data entry fields.
  • Buttons/Hyperlinks/Drop Downs: Hyperlinks to external professional profiles (e.g., LinkedIn).
  • Actions: Basic navigation and link clicks.

3. Projects (*projects.html*)

  • Purpose: Serve as the main portfolio and gallery of student work.
  • Audience: Prospective Students and Industry Professionals.
  • Content: Grid of project cards with images, short descriptions, and filtering options.
  • Data Entry/Validation: No data entry fields.
  • Buttons/Hyperlinks/Drop Downs: Dropdown/select list for filtering by project category (e.g., UX, Dev, Research).
  • Actions: Filtering/Sorting of project cards using JavaScript/jQuery. Navigation to Project Detail page upon click.

4. Events (*events.html*)

  • Purpose: Display a clear, organized timeline of all upcoming group events and meetings.
  • Audience: Current Members and Prospective Students.
  • Content: Event titles, dates, times, and descriptions.
  • Data Entry/Validation: No data entry fields.
  • Buttons/Hyperlinks/Drop Downs: None (information presentation only).
  • Actions: Display of event details using a jQuery-UI Accordion widget.

5. Join/Contact (*contact.html*)

  • Purpose: Collect information from prospective members and provide formal contact details.
  • Audience: Prospective Students and external contacts.
  • Content: A sign-up form, general contact email, and embedded Google Map.
  • Data Entry/Validation: Name, Email (required, validation), Student ID, Interest Area, Message. All fields require client-side validation.
  • Buttons/Hyperlinks/Drop Downs: Submit button and Interest Area dropdown.
  • Actions: Form submission simulation (since CRUD is restricted) after successful client-side validation.

Project Detail Page (Template)

Special Note: This template dynamically loads content after clicking a project on the Projects page. It features an in-depth case study, team member credits, process breakdown, and a modal image gallery (lightbox effect).

💡 Dynamic Functionality Implementation

1. AJAX-powered Featured Project Loader

  • Page: Home Page (*index.html*)
  • Purpose: Loads a curated list of featured projects dynamically upon page load, improving performance and keeping content fresh.
  • Example URL: The New York Times (Observing articles load as you scroll down)

2. jQuery-UI Accordion Widget for Events

  • Page: Events (*events.html*)
  • Purpose: Organizes the event list into collapsible sections, preventing the page from becoming visually overwhelming.
  • Example URL: jQuery UI Accordion Demo

3. Interactive Project Gallery & Filtering

  • Page: Projects (*projects.html*)
  • Purpose: Allows users to smoothly filter the project card grid based on a dropdown selection (e.g., 'Research').
  • Example URL: Dribbble Portfolio (Observe the quick filtering options)

4. Client-Side Form Validation

  • Page: Join/Contact (*contact.html*)
  • Purpose: Provides immediate feedback on the sign-up form to ensure all mandatory fields (especially Email format) are correctly filled.
  • Example URL: LinkedIn Sign-Up Page (Shows real-time validation errors)
